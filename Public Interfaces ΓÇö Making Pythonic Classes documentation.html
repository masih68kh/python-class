Making Pythonic Classes

Public Interfaces¶

Class methods¶

Normally, we call methods on class instances. Occasionally you may have use for a method which can be called on the class itself. In this case, you can use the @classmethod decorator.

When a class method is called on the class, the first implicit argument will be the class (rather than an instance of the class).

This may be used to provide alternate constructors:

import math

class Circle:

    def __init__(self, radius):
        self.radius = radius

    @classmethod
    def from_area(cls, area):
        return cls(math.sqrt(area / math.pi))

Now we can create an instance of Circle by providing an area instead of a radius. Subclasses of Circle may also use this method and the correct subclass will be passed in as cls.

>>> circle = Circle.from_area(176.71458676442586)
>>> circle.radius
7.5

The dict class uses the classmethod decorator to provide the alternate constructor fromkeys. The datetime.datetime class also uses the classmethod decorator to provide the alternate constructors now, fromtimestamp, and fromordinal.

Static methods¶

Occasionally you will have a function related to a class which does not operate on instances of the class at all. If you want to store a function in a class that has no need for self or cls, you can use the @staticmethod decorator.

For example, suppose you wanted to have a function that would convert an angle (in degrees) to a grade. We could put this inside the Circle class like this:

class Circle:

    # ...

    @staticmethod
    def angle_to_grade(angle):
        return math.tan(math.radians(angle))

You would use it like this:

>>> Circle.angle_to_grade(45)
0.9999999999999999

Class Attributes¶

You’ll sometimes see class-level attributes used in Python.

Here’s a class-level attribute that stores all instances of its class:

class Point:

    all_points = []

    def __init__(self, x, y):
        self.x = x
        self.y = y
        self.all_points.append(self)

    def __repr__(self):
        return f"Point(x={self.x}, y={self.y})"

Whenever we make a new Point object, it’ll add itself to all_points:

>>> p = Point(1, 2)
>>> Point.all_points
[Point(x=1, y=2)]
>>> Point.all_points[0] is p
True
>>> q = Point(3, 4)
>>> Point.all_points
[Point(x=1, y=2), Point(x=3, y=4)]
>>> q.all_points
[Point(x=1, y=2), Point(x=3, y=4)]

This is kind of a weird thing to do. We don’t normally mutate class-level attributes. They’re useful for storing things like default values that child classes can override/customize.

Type Hints¶

Python also has type hints which you can use to document your code.

The usual convention for type annotations can be enforced using a static type checker called mypy.

You can install mypy with pip:

$ pip install mypy

Once you have mypy installed, you can run it against code from the command-line.

import math


class Circle:

    def __init__(self, radius=1):
        self.radius = radius

    @classmethod
    def from_area(cls, area):
        return cls(math.sqrt(area / math.pi))

    @property
    def area(self):
        return math.pi * self.radius ** 2

    @property
    def diameter(self):
        return self.radius * 2

    @diameter.setter
    def diameter(self, diameter):
        self.radius = diameter / 2

And run mypy against it:

$ mypy circle.py
Success: no issues found in 1 source file

We didn’t get any output because we don’t have any type annotations.

We can ask mypy to require type annotations by running it in --strict mode:

$ mypy --strict circle.py
circle.py:5: error: Function is missing a type annotation
circle.py:9: error: Function is missing a type annotation
circle.py:13: error: Function is missing a return type annotation
circle.py:17: error: Function is missing a return type annotation
circle.py:21: error: Function is missing a type annotation
Found 5 errors in 1 file (checked 1 source file)

We can add type hints to all arguments and all return values to fix this:

import math


class Circle:

    def __init__(self, radius: float = 1):
        self.radius = radius

    @classmethod
    def from_area(cls, area: float) -> 'Circle':
        return cls(math.sqrt(area / math.pi))

    @property
    def area(self) -> float:
        return math.pi * self.radius ** 2

    @property
    def diameter(self) -> float:
        return self.radius * 2

    @diameter.setter
    def diameter(self, diameter: float) -> None:
        self.radius = diameter / 2

Note that we need to put ‘Circle’ in quotes because Circle isn’t valid Python code before the Circle class has been defined.

We could also do this by import __future__.annotations to delay evaluation of annotations, as will be the default in Python 4.0:

from __future__ import annotations
import math


class Circle:

    def __init__(self, radius: float = 1):
        self.radius = radius

    @classmethod
    def from_area(cls, area: float) -> Circle:
        return cls(math.sqrt(area / math.pi))

    @property
    def area(self) -> float:
        return math.pi * self.radius ** 2

    @property
    def diameter(self) -> float:
        return self.radius * 2

    @diameter.setter
    def diameter(self, diameter: float) -> None:
        self.radius = diameter / 2

Documentation¶

Functions, modules, and classes can all have docstrings. If the very first statement in a function, module, or class is a string literal on its own, it is considered a docstring.

Let’s document our whole circle module:

"""Tools for representing circle in Python."""
from __future__ import annotations
import math


class Circle:

    """A circle with radius, diameter, and area."""

    def __init__(self, radius: float = 1):
        self.radius = radius

    @classmethod
    def from_area(cls, area: float) -> Circle:
        """Return a new Circle with a radius appropriate for the given area."""
        return cls(math.sqrt(area / math.pi))

    @property
    def area(self) -> float:
        """Return the area of the circle."""
        return math.pi * self.radius ** 2

    @property
    def diameter(self) -> float:
        """The diameter is radius * 2."""
        return self.radius * 2

    @diameter.setter
    def diameter(self, diameter: float) -> None:
        self.radius = diameter / 2

Docstrings can be used on functions, classes, and modules. They must be the first string in each of these.

We can see all this documentation using help:

>>> import circle
>>> help(circle)

Or by using pydoc:

$ python -m pydoc circle

Docstrings ideally should note what exceptions various methods/functions can raise and what attributes are available on classes.

PEP 257 talks about how to write docstrings to document your code.
